# CDEK - Новостной портал (тестовое задание)

Универсальный компонент новостного блока для внутреннего портала CDEK. Получает данные с API, отображает список новостей с пагинацией, поддерживает несколько визуальных вариантов и состояний (загрузка, ошибка, пустой список).

> **Стек:** React 19 · TypeScript · Vite 8 · Tailwind CSS v4 · SCSS · Axios · React Router v7 · Tabler Icons

## 🧱 Архитектура

Проект построен по методологии **Feature-Sliced Design (FSD)** - это даёт чёткое разделение ответственности и изолирует слои друг от друга, что важно для масштабируемости портала.

```
src/
├── app/              - точка входа, провайдеры (router), глобальные стили, типы
├── pages/            - страницы (MainPage с code-splitting через React.lazy)
├── widgets/          - крупные самодостаточные блоки
│   ├── NewsBlock/    - ★ главный компонент задания (UI + хук useNewsFeed)
│   ├── Page/         - лейаут страницы (Header + content + Footer)
│   ├── Header/       - шапка с логотипом
│   ├── Footer/       - подвал со ссылками автора
│   └── PageLoader/   - полноэкранный лоадер для Suspense fallback
├── entities/         - бизнес-сущности
│   └── News/         - типы, API-методы и UI-карточки (NewsCard / Default / Compact / ArticleDetails)
└── shared/           - переиспользуемая «инфраструктура»
    ├── api/          - настроенный axios-инстанс
    ├── const/        - роутинг
    ├── lib/          - formatDate, getImageUrl, useSwipe, useDragOffset
    ├── styles/       - шрифт TT Fellows
    └── ui/           - атомы UI: Badge, Divider, LazyImage, Skeleton, Loader,
                      FadeTransition, MetaText, StatCounter, Icon
```

## 💡 Принятые решения

### Универсальность через `type`

Главный компонент `<NewsBlock type="..." title="..." />` принимает один из трёх вариантов:

- `company` - карточки с картинкой у каждой новости (макет «Новости компании»)
- `rubric` - компактные карточки, картинка только у первой (макет «Рубрика Бизнес»)
- `important` - третий визуальный вариант с форматом даты «день недели, число месяц», демонстрирует пустое состояние

Все три варианта рендерит **один и тот же компонент**, переключение между макетами и эндпоинтами происходит через пропс. Это полностью соответствует требованию ТЗ о «универсальном компоненте, который будет использоваться на нескольких страницах».

### Разделение карточек: `NewsCardDefault` и `NewsCardCompact`

Вместо одного компонента с десятком условных пропсов карточка разделена на две реализации, выбираемые в `<NewsCard type="default | compact" />`. Это упрощает чтение и тестирование каждого варианта макета по отдельности.

### Состояние и кэширование в `useNewsFeed`

Кастомный хук инкапсулирует всю логику запроса:

- хранит `page`, `data`, `isLoading`, `error`;
- кэширует ответы в `useRef<Map<page, response>>` - повторное обращение к уже посещённой странице **не делает сетевого запроса**;
- очищает ошибку при успешной загрузке.

Сам компонент `NewsBlock` остаётся «view-слоем» и не знает о деталях fetch-а.

### Стилизация - Tailwind CSS v4

Используется новый синтаксис `@theme` в `tailwind.css` - все цвета (`--color-text-secondary`, `--color-text-primary`, бейджи и т.д.) объявляются как CSS-переменные и доступны и из утилитных классов, и из инлайновых стилей. Это даёт единый источник правды для дизайн-токенов.

Для модульных стилей лоадера и шрифта **TT Fellows** дополнительно используется SCSS.

### Оси отзывчивости

Адаптивность реализована utility-first классами Tailwind: `max-md:`, `max-xl:`. На мобильных устройствах:

- блоки растягиваются на всю ширину (`max-md:w-full`);
- `NewsCardDefault` перестраивается из горизонтальной в вертикальную раскладку;
- картинка скрывается у вторичных карточек (`max-md:hidden`);
- уменьшаются паддинги (`max-md:p-4`).

### Обработка состояний

В компоненте чётко разделены 4 ветки рендера:

1. **Загрузка** → `<NewsBlockSkeleton />` (полная имитация раскладки).
2. **Ошибка** → блок с заголовком «Ошибка загрузки новостей» и текстом ошибки.
3. **Пусто** → плейсхолдер с картинкой и подписью «Новых новостей нет».
4. **Данные есть** → список карточек + пагинация.

Кнопки «Назад» / «Вперёд» используют семантичный `<button disabled>` с `aria-label`, неактивны на крайних страницах (`page <= 1`, `page >= totalPages`).

## ✅ Реализованные дополнительные требования

Все 4 пункта из секции «Дополнительные требования (будет плюсом)» ТЗ выполнены:

| Требование | Реализация |
|---|---|
| **Кэширование** | `useRef<Map<number, NewsFeedResponse>>` внутри `useNewsFeed` - посещённые страницы не перезапрашиваются. У `LazyImage` дополнительно есть свой `Set<string>`-кэш загруженных URL, чтобы при повторном маунте картинки не «мигали» снова. |
| **Ленивая загрузка изображений** | Кастомный компонент `<LazyImage />` на `IntersectionObserver`: атрибут `src` подставляется только когда картинка попадает во вьюпорт. |
| **Анимации** | Компонент-обёртка `<FadeTransition />` (плавное `opacity + translateY`). Каждая карточка появляется со ступенчатой задержкой `index * 100ms`, при смене страницы анимация перезапускается через `transitionKey={page}`. Сами блоки тоже появляются последовательно (delay 100/200/300ms). |
| **Типизация** | Полная типизация на TypeScript: интерфейсы ответа API (`NewsFeedResponse`, `News`, `NewsCover`, …), пропсы всех компонентов, дискриминированные union-типы у `Badge` и `useNewsFeed`. `tsconfig` со `strict: true`. |

## ✨ Что реализовано сверх ТЗ

Помимо обязательных и бонусных требований, добавлено:

- **Feature-Sliced Design** - слоистая архитектура (`app / pages / widgets / entities / shared`) с правилом изоляции слоёв и единой точкой реэкспорта через `index.ts` в каждом сегменте.
- **Свайпы** - кастомные хуки `useSwipe` (с порогом 50px) и `useDragOffset` (визуальная отдача до ±10px при перетаскивании). На тач-устройствах между страницами можно листать жестами, а кнопки оставлены как fallback для десктопа.
- **Просмотр статей в модальном окне** - реализован через универсальный компонент Modal (на базе нативного `<dialog>` с блокировкой скролла и анимациями) и типизированный хук `useModal<T>`. Содержимое вынесено в сущность `ArticleDetails` с `ArticleSkeleton` для имитации загрузки текста.
- **Code-splitting** - `MainPage` загружается лениво (`MainPage.async.tsx` через `React.lazy + Suspense`), под Suspense fallback подложен полноэкранный спиннер `PageLoader`.
- **Изолированный роутинг** - `react-router-dom` с типизированной конфигурацией (`Record<AppRoutes, RouteObject>`) и `RouteErrorBoundary` для перехвата ошибок маршрута.
- **Дизайн-токены через Tailwind v4** - палитра, фоны, цвета бейджей вынесены в `@theme` как CSS-переменные.
- **Линтеры из коробки** - ESLint (с `react-hooks`, `unused-imports`, `import`) и Stylelint, плюс готовые npm-скрипты для `--fix`.
- **Header / Footer** - фиксированная плавающая шапка с логотипом CDEK и подвал с моими ссылками.
- **Доступность** - `aria-label` на кнопках пагинации, `alt` у всех `<img>`, семантичная разметка (`<h2>`, `<h3>`, `<button type="button">`).
- **Path-алиасы** - `@/*` → `src/*` в `tsconfig.json` и `vite.config.ts` (никаких `../../../`).

## 🔌 API

Используются два эндпоинта (см. `src/entities/News/api/newsApi.ts`):

```
GET /api/v1/news/feed/company/short?perPage=3&page=1   - обычная лента
GET /api/v1/news/feed/company/empty?perPage=3&page=1   - имитация пустой ленты
```

Базовый URL берётся из `VITE_API_URL` и инжектится в код через `define.__API_URL__` в `vite.config.ts`.
